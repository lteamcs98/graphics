Here's how it works

Easy Setup

Install our NodeJS application on your web server in a few simple steps.

Write a Challenge

Challenges consist of a function name, test input and a challenge solution. Construct them as JSON objects and upload them in batches to the website.

Distribute to Students

Embed the interactive editor associated with the challenge as an iframe on any web page.

View Results

Track students' progress and manage existing challenges through the application's GUI.

Writing the JSON file

To upload a challenge, you must first write a JSON formatted file containing the following information about the challenge:
* the problem
* the title of the challenge
* the test inputs to the challenge
* the correct test outputs
* the function names
* the function headers

In the JSON file, the fields above will be called the following:
* problem
* title
* inputArray and inputs
* outputArray and outputs
* functionNames
* functionHeaders

Let's look at the following challenge JSON file below, "Average of Three":


The order of the fields doesn't matter -- notice how outputArray comes before inputArray in this example. The name-value assignment for JSON and the nesting of arrays are the most crucial components of the JSON file.

A single JSON object is signified by the curly braces {} that follow the square brackes. The information enclosed in these curly braces constitute the test cases for a single function in this challenge. In the case of this "Average of Three" example, there is only one function to test.

In this case there are two important components to test a single function: inputs and function name. Following "inputs" we see an array of input cases. Above, we can see that there are two input cases (because there are two arrays in the "inputs" array). The first array means that the first test case will provide three parameters (1, 2, 3) to the function being tested. Similarly, we can see that the second test case consists of passing the parameters 50, 60, and 70 into the function being tested.

Lastly, the "functionName" property within the object signifies which function this JSON object provides test cases for. So in our example the function we are testing is fittingly named "averageOfThree".

The complimentary piece to the "inputArray" is the "outputArray" which is directly above it. Again, we have an array that contains a JSON object for each test case. Since we have only one function to test, we see only a single test case. This time we have an array that contains the correct output for the corresponding test case in the "inputArray". So for the case in which the input is 1, 2, 3 we see that the correct output is the value 2. And just like the "inputArray", there is the "functionName" property that signifies which function these outputs belong to.

Upload to the Site

To upload your JSON file(s) to the website, navigate to "Add New Challenges", which is available on your admin homepage. Find your JSON file(s) after clicking "Browse" and press "Upload."

Completing Challenges

1. Navigate to challenge page
2. Type code to solve challenge in the editor
3. Submit code to save and view results



Submitting a Challenge

Once you navigate to a page with a challenge you will notice a window with four key components: the editor, console, unit testing dropdown, and the results pane.

Editor

This is where you write the JavaScript code to solve the coding challenge.

Console

All console output will appear in this window (use console.log in your own code to print)

Unit Testing

If you only wish to test a portion of your code without submitting to save, use this dropdown menu to choose the function that you would like to test. Now when you click the submit button only this function will be tested. Keep in mind that unless the option for "all" tests is selected, then your code will not be submitted to the server and saved.

Results

Student Version: Once you click submit, this pane will show you the test cases run to evaluate your code. If you pass a test then it will appear in green. If your code fails a test then the input, correct output, as well as your output will appear in red in this window. Make your corrections and resubmit!
Instructor Version: To see the results of all of your students, navigate to the results page for the specific challenge. You can get to this page by clicking the "results" tab in the admin portal.
